## Phase 5 — Écosystème Skills

**Critère de sortie :** les skills publics Clawvis sont publiés et découvrables hors du repo.

> Phase post-lancement. Prérequis : Phase 4 complète, au moins 5 skills publics stables et documentés.

### 5A — Qualité & critères marketplace

Un skill est "marketplace-ready" si :
- Il a un `README.md` avec description, exemples d'usage, prérequis
- Il est testé via `skill-tester` (coverage ≥ 80% des cas nominaux)
- Il n'a pas de dépendances d'instance (pas de secrets, pas de paths hardcodés)
- Il a un `CHANGELOG.md` ou section changelog dans son README
- Son nom suit la convention `<domaine>-<action>` (ex: `kanban-implementer`, `brain-pulse`)

### 5B — Publication OpenClaw marketplace

| # | Action | Détails |
|---|--------|---------|
| 5B.1 | Identifier les skills candidats (publics, génériques, stables) | Audit `clawvis/skills/` |
| 5B.2 | Ajouter metadata marketplace à chaque skill (`skill.json` : author, version, tags, description) | Format à définir avec OpenClaw |
| 5B.3 | Soumettre les skills au registre OpenClaw | Process de review OpenClaw |
| 5B.4 | Automatiser la publication via CI au push d'un tag `skill-v*` | `.github/workflows/publish-skill.yml` |

### 5C — Publication Claude MCP marketplace (optionnel)

Les skills Clawvis ne sont pas des MCP tools nativement. Une conversion est nécessaire :

| # | Action | Détails |
|---|--------|---------|
| 5C.1 | Évaluer quels skills ont un mapping MCP naturel (outils CRUD, recherche, actions) | Audit fonctionnel |
| 5C.2 | Créer un wrapper `skills/mcp-bridge/` qui expose les skills compatibles comme MCP tools | Anthropic SDK |
| 5C.3 | Soumettre au Claude MCP marketplace | Process review Anthropic |

> **Note :** L'effort de conversion MCP est significatif. À prioriser seulement si la communauté Claude est une cible stratégique.

### Workflow de création d'un nouveau skill

```
1. Créer le dossier dans clawvis/skills/<name>/ ou instances/<name>/skills/<name>/
2. clawvis skills sync   ← auto-symlink vers ~/.openclaw/skills/
   (ou automatique via git hook post-merge)
3. Tester via skill-tester
4. Si public + stable → ajouter metadata skill.json pour marketplace
```

## Robustification production — Checklist de déploiement

> Référence : `docs/adr/0004-production-deployment-pitfalls.md` (8 pitfalls documentés)

Ces points doivent être vérifiés à chaque nouveau déploiement ou upgrade :

### Avant le premier déploiement d'une instance

- [ ] **Symlink** : si `instances/<name>` est un symlink, ajouter des volume overrides explicites dans `docker-compose.override.yml` pour chaque service qui lit ce chemin
- [ ] **`hub_settings.json`** : mettre à jour `projects_root` avec le chemin réel des projets sur l'hôte (default = placeholder invalide)
- [ ] **HOME env** : tout container utilisant `Path.home()` doit avoir `HOME=<user_home>` dans son environment + volume mount sur les paths home nécessaires

### Avant de recharger nginx

- [ ] Toujours `export` les variables avant `envsubst` (variables non exportées = vides dans le processus enfant)
- [ ] Scoper `envsubst` avec la liste explicite : `envsubst '${HUB_ROOT} ${LAB} ${OPENCLAW_GATEWAY_TOKEN}'`
- [ ] Vérifier `nginx -t -c <conf>` — l'avertissement sur le pid est un faux positif, les autres erreurs sont réelles
- [ ] Après `kill -HUP`, vérifier `nginx -error.log` pour confirmer le rechargement

### Après tout changement de code source

- [ ] Rebuilder explicitement : `docker compose build <service>`
- [ ] Forcer la recréation : `docker compose up -d --force-recreate <service>`
- [ ] Vérifier que le JS déployé contient bien les strings attendus (ex: `docker exec hub grep -r "expected-string" /usr/share/nginx/html/`)

### Nginx — location blocks requis pour Hub SPA

- [ ] `location /assets/` — AVANT `/hub/` (Vite absolute asset paths)
- [ ] Regex pour root statics : `location ~ ^/(clawvis-mascot\.svg|favicon\.ico|favicon\.png)$`
- [ ] Redirects pour SPA sub-paths accessibles directement : `location = /chat/ { return 301 /hub/#/chat; }`
- [ ] Vérifier les doublons AVANT d'ajouter un location block — nginx refuse les doublons au démarrage
